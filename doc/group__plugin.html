Detailed Description

Writing an LV2 Plugin.

Plugin descriptors are registered on the stack at the global scope. For example...

using MyPlugin = lvtk::Plugin<MyInstance>;
static MyPlugin my_plugin (
LVTK_SILENCE_URI, //< MyPlugin's URI String
{
LV2_URID__map, //< List of required host features
LV2_WORKER__schedule
}
);

If you are using this library in a host, include individual headers from the ext directory.

Same policy for the interface directory, including anything there will result in a mess of compiler errors. Interfaces are for implementing features on a plugin instance only
